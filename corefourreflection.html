Core4 Reflections!

In my programming experience, I've been taught the Core4, which are:

Conditionals, which use booleans (true-false statements) to execute different actions based on the action;
Loops, which make executed actions repeat themselves until a certain condition is reached;
Variables, which are used to store certain data to use in a program, signified by a symbolic name called an identifier;
and Functions, which are a section of a program that perform a specific task.

Differences in Python and Scratch

The pictures above show coding in Scratch and Python. Although there are definite similarities in both programs (such as the ability to use loops, variables, and functions effectively), Scratch is very limited in scope and there is only so much one can do without going to great lengths to code something more personalized. Python, on the other hand, is a very powerful multipurpose programming tool, and can be used to create more personalized and more complex codes.

Analogies of the Core4

A real-life example for conditionals (if-else statements) could be that if it is sunny outside, you will walk. Else, you will stay home.
For loops, it can be understood with daily routines: until you reach Friday, for example, you will keep on working.
Variables are like keeping score in a game; if a certain condition is met, the score is told to go up by one, just like how a variable can change if a condition is met.
Functions are definitions for certain actions, and is like how instead of calling something a four legged, furry mammal, you can call it a dog.
